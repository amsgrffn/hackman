# Hackman — Ghost Theme

A custom Ghost 6.0+ blog theme built with Handlebars, PostCSS, and Rollup.

## Philosophy

- **Performance first** — Lean code, no bloat. Reuse existing code, don't reinvent the wheel.
- **Responsive first** — Everything must work on mobile (iPhone-sized) first. Design mobile-up.
- **Security** — XSS-safe DOM methods everywhere. Never use `innerHTML` with user data.
- **htmx first** — Reach for htmx before writing custom JavaScript.
- **Simplicity** — Minimal dependencies, simplest solution wins. KISS. Do not over-engineer.
- **Accessible** — Follow W3C accessibility standards (WCAG). Use semantic HTML, proper ARIA attributes, focus management, and sufficient color contrast.

## Commands

```bash
npm run dev       # Development mode with livereload (watches .hbs, .js, .css)
npm run build     # Production build (minified + optimized)
npm run zip       # Build + zip for theme upload
npm run test      # Run gscan theme validation (builds first via pretest)
```

## Project Structure

```

│   ├── base/             # Reset, variables, typography, layout
│   ├── components/       # UI components (cards, forms, buttons)
│   ├── ghost/            # Ghost-specific overrides
│   ├── pages/            # Page-specific styles
│   ├── utilities/        # Animations, responsive, dark mode
│   └── index.css         # Main entry (imports all above)
├── js/
│   ├── index.js          # Main JS entry point
│   ├── carousel.js       # Carousel logic
│   ├── carousel-continuous.js

├── built/                # Compiled output (do not edit)
└── media/                # Images, avatars, GIFs

partials/
├── cards/                # Post card variants (article, bookmark, featured, note, quote, video)
├── carousel/             # Featured posts carousel
├── icons/                # SVG icon partials
├── layout/               # Header, footer, sidebar
└── utilities/            # Mobile menu, post dropdown

members/                  # Membership pages (signin, signup, account)

```

## Build System

- **Bundler**: Rollup — entry point is `assets/js/index.js`, outputs to `assets/built/`
- **CSS**: PostCSS with `postcss-import` and `postcss-preset-env`
- **JS**: Babel (`@babel/preset-env`) transpilation, Terser minification
- **Output format**: IIFE with source maps
- **Dev server**: LiveReload watches `.hbs`, `.js`, `.css` files

## Code Guidelines

### Handlebars Templates

- Use Ghost's `{{#foreach}}`, `{{#if}}`, `{{content}}` helpers — check Ghost docs before adding custom logic.
- Reuse partials from `partials/` instead of duplicating markup.
- Keep templates flat. If a partial grows beyond ~50 lines, consider splitting it.

### CSS

- Add new styles to the appropriate subdirectory in `assets/css/` (base, components, pages, utilities).
- Import new files in `assets/css/index.css`.
- Use CSS custom properties defined in `assets/css/base/` for colors, spacing, and typography.
- Mobile-first media queries — start with small screens, add `min-width` breakpoints up.

### JavaScript

- Prefer htmx attributes in templates over custom JS.
- When JS is necessary, add to existing files or create focused modules in `assets/js/`.
- Use `document.createElement`, `textContent`, `setAttribute` — never `innerHTML` with dynamic data.
- Use event delegation where possible to reduce listeners.
- Keep DOM queries minimal; cache references when reused.

### Accessibility

- Use semantic HTML elements (`<nav>`, `<main>`, `<article>`, `<button>`, etc.).
- All images need `alt` attributes. Decorative images use `alt=""`.
- Interactive elements must be keyboard-navigable with visible focus styles.
- Use `aria-label` / `aria-describedby` only when semantic HTML alone is insufficient.
- Ensure color contrast meets WCAG AA minimum (4.5:1 for text, 3:1 for large text).

### General

- Never edit files in `assets/built/` — they are generated by the build.
- Run `npm run test` (gscan) before finalizing changes to catch Ghost theme errors.
- Keep dependencies minimal. Check if PostCSS/Rollup/existing code already solves the problem before adding a package.

## Deployment

The theme auto-deploys on push to `main`/`master` via `.github/workflows/deploy-theme.yml` using TryGhost's deploy action. Requires `GHOST_ADMIN_API_URL` and `GHOST_ADMIN_API_KEY` secrets configured in GitHub.
